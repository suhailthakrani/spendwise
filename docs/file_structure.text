SpendWise — Proposed File Structure
====================================
Goal: offline-first expense tracker. Feature-first layout + shared data layer.
State: Riverpod. Persistence: local DB (Drift recommended). No extra layers unless needed.

Current state (today)
---------------------
lib/
  main.dart
  app/app.dart
  core/          constants, router, theme, utils, widgets
  data/          models, providers/spendwise_data_provider.dart (mock singleton)
  features/      shell, dashboard, expenses, categories, reports, budget, search, account

What we are NOT adding (on purpose)
-----------------------------------
- No separate domain/ folder with use-case classes per action
- No service per entity (repos are enough for CRUD)
- No global event bus
- No code generation for models unless JSON/API is added later
- No feature/data/presentation subfolders deeper than screens + providers

Migration order (suggested)
---------------------------
1. Add Riverpod + Drift, database_provider, tables
2. One repository at a time (expense first), replace mock reads
3. Wire feature providers, then screens
4. Preferences persistence
5. Reports/aggregations in report_repository
6. Export service last
